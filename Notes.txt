1. Why is database-level validation better than only checking in controller?

Database level validation is better because it guarantees data integrity at the source.
Controller checks are useful for user feedback, but they can be bypassed or missed. 
With constraints and validations at the database level, you ensure consistency, security, and 
reliability across all applications that interact with the data.

6. Why Use Middleware for Authentication?
Using middleware for authentication centralizes the logic, making it reusable across multiple routes.
It also keeps controllers clean and focused on business logic, while the middleware handles security concerns.
7. Error Handling Middleware
In Express.js, error handling middleware is defined with four parameters: (err, req, res, next). It catches errors thrown in routes or controllers and allows you to send a consistent error response to the client. This helps in maintaining a clean separation of concerns and ensures that all errors are handled gracefully.
8. Benefits of Using Middleware
- Reusability: Middleware functions can be reused across different routes, reducing code duplication.
- Separation of Concerns: Middleware allows you to separate different aspects of your application (e.g., authentication, logging, error handling) into distinct functions.
- Flexibility: You can easily add, remove, or modify middleware functions without affecting the core
9. Best Practices for Middleware
- Keep middleware functions focused on a single task.
- Use descriptive names for middleware functions to indicate their purpose.
- Handle errors within middleware and pass them to error handling middleware when necessary.
- Avoid performing heavy computations in middleware to prevent blocking the event loop.
10. Multi-Tenants Architecture
In a multi-tenant architecture, a single instance of the application serves multiple tenants (clients). Each tenant's data is isolated and secure, ensuring that one tenant cannot access another tenant's data. This can be achieved through various strategies such as using separate databases, schemas, or tables for each tenant. Multi-tenancy allows for efficient resource utilization and easier maintenance while providing a scalable solution for serving multiple clients.
11. CORS (Cross-Origin Resource Sharing)

Cross-Origin Resource Sharing (CORS) is a security feature implemented by browsers to restrict web applications from making requests to a different domain than the one that served the web page. It is a mechanism that allows servers to specify who can access their resources and how they can be accessed. CORS is important for preventing malicious websites from making unauthorized requests to your server, 
but it can also be configured to allow legitimate cross-origin requests when necessary.
12. Hooks in Express.js
Hooks in Express.js are functions that can be executed at specific points in the request-response cycle. They allow you to perform actions before or after certain events, such as before a route handler is executed or after a response is sent. Hooks can be used for tasks like logging, modifying request or response objects, or implementing custom behavior. They provide a way to extend the functionality of your application without modifying the core logic of your routes or controllers.
